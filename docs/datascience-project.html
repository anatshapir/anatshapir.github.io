SECTION 01

מהי למידת מכונה?

למידת מכונה היא תחום שעוסק במתן יכולת למחשבים ללמוד מבלי שיתוכנתו במפורש
— ארתור סמואל, 1959

💡 במילים פשוטות:

במקום לכתוב חוקים מפורשים, אנחנו מראים למחשב דוגמאות והוא לומד בעצמו את הדפוסים.

🧠

לומד מדוגמאות

לא צריך לכתוב כל חוק ידנית – המכונה מזהה דפוסים לבד

📈

משתפר עם נתונים

ככל שיותר נתונים = בדרך כלל תוצאות טובות יותר

🔄

מסתגל

לומד דפוסים חדשים כשהנתונים משתנים

תכנות רגיל vs למידת מכונה ההבדל המרכזי

📝 תכנות רגיל

📊 Data + 📏 Rules → 💡 Answer

המתכנת כותב את החוקים ידנית. המחשב רק מפעיל אותם.

🤖 למידת מכונה

📊 Data + 💡 Answers → 📏 Rules

המחשב לומד את החוקים בעצמו מתוך הנתונים והתשובות.

🔑

ההבדל: בתכנות רגיל אנחנו כותבים חוקים. ב-ML המחשב מגלה את החוקים מדוגמאות. כשהבעיה מורכבת מדי לחוקים ידניים (כמו זיהוי פרצופים) – ML עדיף!

📧 דוגמה: זיהוי ספאם למה ML עדיף?

📝 גישה רגילה – חוקים ידניים

rules.py
if "free money" in email: spam = True if "click here" in email: spam = True if "winner" in email: spam = True # ... עוד מאות חוקים? # 😫 הספאמרים משנים נוסח!

🤖 גישה ML – לומד מדוגמאות

ml_approach.py
# מאמנים על אלפי מיילים מתויגים model.fit(emails, labels) # המודל לומד לבד אילו דפוסים # מעידים על ספאם! # ✅ מסתגל לניסוחים חדשים # ✅ מזהה דפוסים מורכבים
💬

מתי בוחרים מה?
חישוב ממוצע ציונים → תכנות רגיל (חוק פשוט וברור)
זיהוי חתולים בתמונות → ML (איך כותבים "חוק" לזיהוי חתול? 🐱)
המלצות סרטים → ML (דפוסים מורכבים בטעם אישי)

SECTION 02

סוגי למידת מכונה

שלושה סוגים עיקריים
🏷️

למידה מפוקחת

Supervised Learning

לומד מדוגמאות עם תשובות (תוויות)

✅ יש תוויות
📌 ניבוי מחירים, זיהוי ספאם, אבחון רפואי

🔍

למידה לא מפוקחת

Unsupervised Learning

מוצא דפוסים בלי תשובות

❌ אין תוויות
📌 פילוח לקוחות, זיהוי אנומליות

🎮

למידה מחוזקת

Reinforcement Learning

לומד מניסוי וטעייה עם תגמולים

🎯 פידבק (שכר/עונש)
📌 משחקים, רובוטים, רכב אוטונומי

📌

בקורס שלנו נתמקד בלמידה מפוקחת – הסוג הנפוץ ביותר בתעשייה.

סיווג vs רגרסיה למידה מפוקחת

שני סוגי בעיות בלמידה מפוקחת – לפי סוג התשובה

🏷️ סיווג (Classification)

התשובה היא קטגוריה

האם זה ספאם? → כן / לא
איזה חיה? → חתול / כלב / ציפור
האם יעבור? → עובר / נכשל

📈 רגרסיה (Regression)

התשובה היא מספר רציף

מה מחיר הדירה? → 2,500,000 ₪
מה הטמפרטורה? → 28.5°C
כמה זמן? → 3.2 שעות

🧪 תרגלו – סיווג או רגרסיה?

ניבוי מחיר דירה → רגרסיה (מספר)
זיהוי תמונת כלב/חתול → סיווג (קטגוריה)
ניבוי ציון במבחן → רגרסיה (מספר)
אבחון מחלה (חולה/בריא) → סיווג (קטגוריה)
ניבוי מספר צפיות ביוטיוב → רגרסיה (מספר)
סיווג מייל (ספאם/לא) → סיווג (קטגוריה)
🎯

כלל פשוט: אם התשובה היא "כמה?" → רגרסיה. אם התשובה היא "מה?" / "האם?" → סיווג.

SECTION 03

מושגי יסוד

המילון שחייבים לדעת

Feature (מאפיין)

משתנה קלט – מידע שעוזר לחזות. למשל: גודל דירה, מספר חדרים, קומה

Target / Label (יעד)

מה שרוצים לחזות. למשל: מחיר הדירה, האם ספאם

Sample (דגימה)

שורה אחת בנתונים – דוגמה אחת. למשל: דירה ספציפית עם כל הפרטים שלה

Model (מודל)

האלגוריתם שלומד את הדפוסים ויודע לחזות על נתונים חדשים

Training (אימון)

התהליך שבו המודל לומד מהנתונים

Prediction (חיזוי)

כשהמודל מקבל נתון חדש ומנחש את התשובה

import pandas as pd import numpy as np # יצירת נתוני תלמידים np.random.seed(42) n = 50 # 50 תלמידים hours = np.random.uniform(1, 10, n) prev = np.random.uniform(40, 100, n) passed = ((hours * 5 + prev * 0.5) > 70).astype(int) df = pd.DataFrame({ 'Hours_Studied': hours.round(1), 'Previous_Grade': prev.round(1), 'Passed': passed # 1=עבר, 0=נכשל }) print(df.head(8))

🏋️ שלב 4-5: אימון והערכה KNN

מאמנים מודל KNN – השכנים הקרובים
step4_train.py
from sklearn.neighbors import KNeighborsClassifier from sklearn.metrics import accuracy_score # שלב 4: יצירה ואימון model = KNeighborsClassifier(n_neighbors=5) model.fit(X_train, y_train) print('✅ Model trained!') # שלב 5: חיזוי על הטסט + הערכה y_pred = model.predict(X_test) accuracy = accuracy_score(y_test, y_pred) print(f'Accuracy: {accuracy:.0%}')
▼
OUTPUT ▾ ✅ Model trained! Accuracy: 90% ← 9 מתוך 10 תלמידים נוחזו נכון!

🏘️ איך KNN עובד?

עבור תלמיד חדש, המודל מסתכל על 5 השכנים הכי קרובים (K=5) בנתוני האימון. אם רובם עברו → חוזה "עובר". אם רובם נכשלו → חוזה "נכשל".

📏

Accuracy = 90% פירושו: מתוך 10 תלמידים בסט הבדיקה, המודל צדק ב-9 מקרים.

SECTION 05

Overfitting ו-Underfitting

שתי בעיות נפוצות שחשוב להכיר

📚 Overfitting – התאמת יתר

המודל "שינן" את נתוני האימון במקום ללמוד דפוסים כלליים.

כמו תלמיד שמשנן תשובות בעל-פה אבל לא מבין – נכשל בשאלות חדשות.

Train: 99%   Test: 65%   ⚠️ פער!

😴 Underfitting – התאמת חסר

המודל פשוט מדי – לא הצליח ללמוד את הדפוסים.

כמו תלמיד שלא למד בכלל – נכשל גם בשאלות קלות.

Train: 55%   Test: 50%   ⚠️ שניהם נמוכים!

✅ Good Fit – מודל טוב

Train: 88%   Test: 85%   ✅ קרובים!

ביצועים דומים על אימון ובדיקה – סימן שהמודל הכליל (generalized) טוב.

🔑

איך מונעים Overfitting? יותר נתונים, מודל פשוט יותר, הסרת מאפיינים לא רלוונטיים. הסימן: פער גדול בין Train ל-Test.
