SECTION 01

מהו KNN?

K-Nearest Neighbors – "תגיד לי מי חבריך ואומר לך מי אתה"

💡 הרעיון בפשטות:

כדי לסווג נקודה חדשה, מסתכלים על K השכנים הכי קרובים ומחליטים לפי הרוב.

EXAMPLE: K=5
New point ⭐ has 5 nearest neighbors:
🔵 🔵 🔵 🔴 🔴
3 Blue vs 2 Red → Classify as BLUE!
(majority vote = הצבעת רוב)
✅

יתרונות

פשוט, אינטואיטיבי, אין אימון ממש, עובד טוב על נתונים קטנים

⚠️

חסרונות

איטי על נתונים גדולים, רגיש לסקאלה, צריך לבחור K

🎯

שימושים

סיווג (ספאם, מחלות), המלצות, זיהוי תמונות

📐 חישוב מרחק Euclidean Distance

כדי למצוא "שכנים קרובים" – צריך למדוד מרחק!

📏 מרחק אוקלידי = "קו ישר"

d = √((x₂-x₁)² + (y₂-y₁)²)

distance.py
point_a = (2, 3) point_b = (5, 7) dist = np.sqrt((5-2)**2 + (7-3)**2) print(f'Distance A→B: √(3² + 4²) = √(9+16) = √25 = {dist}') # או בקיצור עם NumPy: print(np.linalg.norm( np.array(point_a) - np.array(point_b)))
▼
OUTPUT ▾ Distance A→B: √(3² + 4²) = √(9+16) = √25 = 5.0
📌

עובד גם ב-3D, 4D, 100D! ככל שיותר מאפיינים – פשוט מוסיפים ממדים לנוסחה.

SECTION 02

נרמול – למה קריטי?

בלי נרמול, KNN נותן תוצאות שגויות!

⚠️ הבעיה: סקאלות שונות

גיל: 20-80 (טווח 60)  |  משכורת: 5,000-50,000 (טווח 45,000)
המשכורת תשלוט על המרחק! הגיל כמעט לא ישפיע.

normalize.py
# בלי נרמול: A = {'age': 25, 'salary': 8000} B = {'age': 55, 'salary': 8500} # גיל שונה! C = {'age': 26, 'salary': 45000} # משכורת שונה! dist_AB = np.sqrt((55-25)**2 + (8500-8000)**2) # = 500.9 dist_AC = np.sqrt((26-25)**2 + (45000-8000)**2) # = 37000!
▼
OUTPUT – בלי נרמול ▾ A→B: 500.9 ← KNN אומר B קרוב! A→C: 37000.0 ← C "רחוק" ❌ אבל A ו-B שונים ב-30 שנה!!! ו-A ו-C דומים בגיל (25 vs 26)... המשכורת שולטת על הכל!
after_norm.py
# אחרי Min-Max Normalization (0-1): # norm = (value - min) / (max - min) dist_AB_norm = 0.502 # עכשיו הגיל כן משפיע! dist_AC_norm = 0.822 # ✅ מאוזן
🔑

כלל זהב: תמיד לנרמל לפני KNN! שיטות: MinMaxScaler (0-1) או StandardScaler (ממוצע=0, סט"ת=1).

🏷️ One-Hot Encoding משתנים קטגוריאליים

KNN עובד רק עם מספרים – אז מה עושים עם "עיר"?

❌ טעות: מספרים רגילים

Tel Aviv → 1
Jerusalem → 2
Haifa → 3
⚠️ KNN חושב: חיפה (3) "רחוקה" מת"א (1)!

✅ פתרון: One-Hot

Tel Aviv → [1, 0, 0]
Jerusalem → [0, 1, 0]
Haifa → [0, 0, 1]
✅ כל עיר שווה, אין "סדר"!

one_hot.py
df_encoded = pd.get_dummies(df, columns=['city']) print(df_encoded)
▼
OUTPUT ▾ city_TelAviv city_Jerusalem city_Haifa 0 1 0 0 1 0 1 0 2 0 0 1 3 1 0 0
📋

סיכום הכנת נתונים ל-KNN: מספרי → נרמול (StandardScaler). קטגוריאלי → One-Hot (get_dummies).

🔀 חלוקת Train / Test למה מחלקים?

❌ בלי חלוקה

מאמנים ובודקים על אותם נתונים = כמו לתת לתלמיד את אותן שאלות שתרגל. לא באמת יודעים אם הבין!

✅ עם חלוקה

מאמנים על 80%, בודקים על 20% שהמודל לא ראה = "מבחן בהפתעה" אמיתי!

split.py
from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.2, # 20% לבדיקה random_state=42, # שחזור stratify=y # שמירת יחס הקטגוריות )
▼
OUTPUT ▾ Training: 48 samples (80%) ← המודל לומד מאלה Testing: 12 samples (20%) ← "מבחן בהפתעה"
📌

stratify=y → שומר יחס זהה של קטגוריות ב-Train וב-Test (למשל 50% תפוחים, 50% תפוזים בשניהם).

SECTION 03

דוגמה מלאה: סיווג פירות 🍎🍊

נבנה מודל KNN לסווג תפוחים ותפוזים לפי משקל וקוטר

▼
df.head(4) ▾ weight diameter fruit 0 159.9 7.08 apple 1 147.2 6.93 apple 2 162.9 6.82 apple ... 30 174.2 7.90 orange 31 223.4 8.93 orange Features: weight, diameter Target: fruit (apple / orange)

⚖️ נרמול + אימון StandardScaler + KNN

normalize_train.py
from sklearn.preprocessing import StandardScaler from sklearn.neighbors import KNeighborsClassifier # נרמול scaler = StandardScaler() X_train_sc = scaler.fit_transform(X_train) # fit+transform על Train! X_test_sc = scaler.transform(X_test) # transform בלבד על Test! # אימון KNN knn = KNeighborsClassifier(n_neighbors=5) knn.fit(X_train_sc, y_train) print('✅ Model trained!')
▼
Before vs After Scaling ▾ Before: weight: [120-240] diameter: [5.5-10] After: weight: [-2, +2] diameter: [-2, +2] ✅ עכשיו שני המאפיינים באותה סקאלה!

⚠️ השגיאה הנפוצה ביותר!

fit_transform על Train בלבד!   transform על Test.
אם עושים fit גם על Test → Data Leakage (דליפת מידע) = תוצאות שקריות!

📏 חיזוי והערכה Accuracy + Confusion Matrix

evaluate.py
from sklearn.metrics import accuracy_score, confusion_matrix y_pred = knn.predict(X_test_sc) accuracy = accuracy_score(y_test, y_pred) print(f'Accuracy: {accuracy:.0%}') cm = confusion_matrix(y_test, y_pred) print(cm)
▼
OUTPUT ▾ Accuracy: 92% Confusion Matrix: Predicted apple orange Actual apple [ 5 1 ] ← 1 תפוח סווג כתפוז orange [ 0 6 ] ← כל התפוזים נכונים!

✅ True Positive / Negative

5 תפוחים זוהו נכון, 6 תפוזים זוהו נכון = 11 הצלחות

❌ Errors

1 תפוח סווג כתפוז (False Negative) = שגיאה אחת
סה"כ: 11/12 = 92%

💡

92% זה לא 100% – וזה נורמלי ורצוי! 100% על Test = חשד ל-Overfitting. בעולם האמיתי תמיד יש חפיפה בין הקבוצות.

SECTION 04

בחירת K אופטימלי

K קטן = רועש, K גדול = מוחלק מדי

K=1 (קטן מדי)

מסתכל רק על שכן אחד → רגיש לרעש
Train: 100% Test: 75%
← Overfitting!

K=50 (גדול מדי)

מסתכל על 50 שכנים → "ממוצע" כולם
Train: 65% Test: 60%
← Underfitting!

find_best_k.py
# בדיקת K=1 עד K=20 for k in range(1, 21): knn = KNeighborsClassifier(n_neighbors=k) knn.fit(X_train_sc, y_train) train_acc = knn.score(X_train_sc, y_train) test_acc = knn.score(X_test_sc, y_test)
▼
OUTPUT ▾ K=1: Train=100% Test=83% ← פער! Overfitting K=3: Train=96% Test=92% ← טוב! K=5: Train=94% Test=92% ← 🏆 Best! (יציב) K=7: Train=92% Test=92% ← עדיין טוב K=15: Train=88% Test=83% ← מתחיל לרדת
🎯

מחפשים K שנותן Test accuracy מקסימלי עם פער קטן מה-Train. בדרך כלל K=3 עד K=9 עובד טוב. עדיף אי-זוגי (למניעת תיקו).

📐 שיטת המרפק (Elbow Method) Error Rate

מסתכלים על שגיאה (Error = 1 - Accuracy) כפונקציה של K
elbow.py
error_rates = [] for k in range(1, 21): knn = KNeighborsClassifier(n_neighbors=k) knn.fit(X_train_sc, y_train) y_pred = knn.predict(X_test_sc) error_rates.append(1 - accuracy_score(y_test, y_pred)) plt.plot(range(1,21), error_rates, 'r-o') plt.xlabel('K'); plt.ylabel('Error Rate')
מה מחפשים בגרף? ▾ Error ↑ │\ │ \ │ \___________ ← "מרפק" = כאן השגיאה מפסיקה לרדת │ \___ │ └────────────────→ K ↑ Best K (בדרך כלל 3-7) 📌 בוחרים את K בנקודת ה"מרפק" – אחריה כבר לא שווה להגדיל K.

# פרי חדש: משקל 180g, קוטר 7.5cm new_fruit = pd.DataFrame({ 'weight': [180], 'diameter': [7.5] }) # 1. נרמול (באותו scaler!) new_scaled = scaler.transform(new_fruit) # 2. חיזוי prediction = knn.predict(new_scaled) proba = knn.predict_proba(new_scaled) print(f'Prediction: {prediction[0]}') print(f'Confidence: {proba}')

📋 תהליך עבודה מלא Pipeline

full_pipeline.py
# 1️⃣ הפרדת מאפיינים ויעד X = df[['weight', 'diameter']] y = df['fruit'] # 2️⃣ חלוקה X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2) # 3️⃣ נרמול (fit על Train בלבד!) scaler = StandardScaler() X_tr_sc = scaler.fit_transform(X_tr) X_te_sc = scaler.transform(X_te) # 4️⃣ אימון knn = KNeighborsClassifier(n_neighbors=5) knn.fit(X_tr_sc, y_tr) # 5️⃣ הערכה print(f'Accuracy: {knn.score(X_te_sc, y_te):.0%}') # 6️⃣ חיזוי new_sc = scaler.transform(new_data) knn.predict(new_sc)
🔑

שימו לב לסדר! חלוקה → נרמול (fit רק Train) → אימון → הערכה → חיזוי. שגיאה בסדר = Data Leakage!
